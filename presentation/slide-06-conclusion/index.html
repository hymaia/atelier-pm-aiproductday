Les concepts clés de Claude Code pour PM

Mapping des concepts
CLAUDE.md
Instructions permanentes
Le contexte que Claude lit à chaque session — votre brief d'onboarding
data/
Votre base de connaissance
Vos fichiers source — calendrier, transcripts, notes, métriques
indexer.py
Pipeline d'indexation
Prépare vos données pour que Claude puisse les interroger efficacement
loaders/
Connecteurs de données
Lisent CSV, TXT, MD, PDF — vos intégrations métier
.claude/skills/
Vos commandes métier
Vos macros IA — exécutables avec /nom-du-skill
Pour aller plus loin
Brancher vos vraies données
Votre calendrier Google, vos tickets Jira, vos interviews — tout ce qui est dans data/ devient contexte pour Claude.
Créer vos propres skills
Tout ce que vous faites de répétitif peut devenir un skill : préparer un comité, rédiger une spec, résumer un sprint.
Partager avec votre équipe
Le repo est partageable. Chaque PM peut avoir son propre contexte dans le même OS — chacun branche ses données.
Repo de l'atelier · github.com/hymaia/atelier-pm-aiproductday
Clonez, forkez, personnalisez — c'est le principe.
